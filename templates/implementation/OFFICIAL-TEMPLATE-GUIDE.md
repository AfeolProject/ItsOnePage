# Official Template Guide

Version: 1.0

Status: IMPLEMENTATION

---

# Purpose

This guide describes how official ItsOnePage templates are created and maintained.

Unlike the Template Specification, this guide focuses on implementation rather than requirements.

---

# Objective

Official templates demonstrate the recommended way of building websites with ItsOnePage.

They serve as reference implementations for the community.

---

# Design Goals

Official templates should be:

- clean
- modern
- responsive
- accessible
- privacy-friendly
- lightweight

They should prioritize simplicity over visual complexity.

---

# Technologies

Official templates should prefer:

- HTML
- CSS
- Minimal JavaScript

No build process should be required whenever reasonably possible.

---

# Images

Responsive images should be provided for:

- Desktop
- Tablet
- Mobile

Images should be optimized for fast loading.

Modern formats such as WebP are recommended.

---

# SEO

Official templates should include:

- title
- description
- canonical URL
- Open Graph
- favicon
- robots
- sitemap

Pages intended only for demonstrations should use:

```text
noindex
nofollow
```

---

# Accessibility

Official templates should support:

- keyboard navigation
- semantic HTML
- sufficient color contrast
- responsive typography

---

# Footer

Official templates should include:

- project name
- version
- copyright

Additional project links may be included when appropriate.

---

# Current Showcase

Templates selected as the Current Showcase may include the official Showcase Footer.

The Showcase Footer is provided by the ItsOnePage project and should not replace the author's own copyright.

---

# Philosophy

Official templates are examples.

They should teach by example rather than by documentation.
