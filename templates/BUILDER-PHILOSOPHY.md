# Basic Builder Philosophy

Version: 1.0

Status: FOUNDATION

---

# Overview

The Basic Builder is a browser-based static website generator.

Its purpose is to help users create portable websites without requiring programming knowledge.

The generated website is the final product.

The Builder itself is only a tool.

---

# Design Principles

The Builder follows several fundamental principles.

## Browser First

Everything that can reasonably happen inside the browser should happen inside the browser.

No application server is required.

No account is required.

No database is required.

---

## Static First

The generated website is static.

It should not require a framework or runtime.

Whenever possible, the generated website consists only of:

```text
HTML

CSS

Images

Local assets
```

---

## Ownership First

The generated website belongs entirely to its owner.

After export:

- no Builder dependency
- no cloud dependency
- no subscription
- no runtime dependency

The owner decides where and how the website is published.

---

## Builder First

The Builder configures websites.

It does not design websites.

Visual identity is provided by Builder Layouts or Custom Templates.

---

## Component First

Every Builder Layout is constructed from reusable components.

Examples:

```text
Background

Panel

Typography

Buttons

Footer

Metadata
```

Every component follows the same specification format.

---

## Rule Engine

The Builder never exposes conflicting options.

Every user action is validated through the Rule Engine.

Example:

```text
IF Website Type = OnePage

THEN

Disable Separate CSS
```

Example:

```text
IF Runtime = Tor / I2P

THEN

Disable JavaScript

Hide SEO
```

---

## Progressive Disclosure

The Builder should display only controls that are currently relevant.

Unavailable options remain visible only when they improve user understanding.

Otherwise they remain hidden.

The interface should never overwhelm the user.

---

## Free Builder

Everything generated by the Basic Builder is free.

The Builder is not a premium template marketplace.

Its purpose is to generate complete websites.

---

## Custom Development

Independent developers remain free to create completely custom templates.

They define:

- HTML
- CSS
- JavaScript
- Architecture
- Licensing
- Pricing
- Distribution

The Basic Builder never restricts independent template development.

---

## Generated Output

The Builder generates complete website files.

Typical output:

```text
index.html

assets/
```

or

```text
index.html

about.html

contact.html

styles.css

assets/
```

The generated website is ready for deployment.

---

## Runtime Profiles

Runtime Profiles define runtime restrictions.

They never define visual appearance.

Initial profiles:

```text
Standard Web

Tor / I2P
```

Future profiles may be introduced without changing Builder architecture.

---

## Website Types

Supported website types:

```text
OnePage

MultiPage
```

Website Type determines output structure.

It does not determine visual design.

---

## Independence

The Builder should not assume:

- hosting provider
- domain registrar
- operating system
- deployment platform
- cloud vendor

Generated websites remain portable.

---

## Long-term Stability

The Builder should evolve without breaking previously generated websites.

Configuration formats may evolve.

Generated websites should remain stable.

---

## Principle

```text
Generate.

Download.

Publish.

Own.

Repeat.
```
